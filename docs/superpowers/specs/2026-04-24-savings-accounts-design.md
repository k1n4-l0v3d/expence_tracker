# Накопительные счета — Design Spec

**Дата:** 2026-04-24  
**Статус:** Approved

---

## Цель

Добавить накопительные счета (копилки) с именованным балансом, операциями пополнения/снятия и целевой суммой накопления. Пополнение уменьшает месячный баланс, снятие — увеличивает.

---

## Модель данных

### Новая таблица `savings_accounts`

| Поле | Тип | Описание |
|---|---|---|
| id | Integer PK | |
| user_id | FK → users | Владелец |
| name | String(100) | Название счёта |
| color | String(7) | HEX-цвет (#rrggbb) |
| icon | String(50) | Bootstrap-иконка, default `bi-piggy-bank` |
| target_amount | Numeric(12,2) nullable | Целевая сумма (null = без цели) |
| is_active | Boolean | По умолчанию True |
| created_at | DateTime | |

### Изменения в существующих таблицах

```sql
ALTER TABLE expenses ADD COLUMN savings_account_id INTEGER NULL
    REFERENCES savings_accounts(id);

ALTER TABLE incomes ADD COLUMN savings_account_id INTEGER NULL
    REFERENCES savings_accounts(id);
```

### Системная категория «Накопления»

При запуске приложения создаётся системная категория (user_id=NULL):
- name = «Накопления»
- icon = «bi-piggy-bank»
- color = «#0d6efd»

Все пополнения счетов используют эту категорию.

### Вычисление баланса счёта

```
balance = SUM(Income.amount WHERE savings_account_id = X)
        - SUM(Expense.amount WHERE savings_account_id = X)
```

---

## Операции

### Пополнение счёта (deposit)
1. Создаёт `Expense(category_id=savings_cat_id, savings_account_id=X, amount, date, description, is_planned=False, is_spent=True)`
2. Баланс месяца уменьшается автоматически (расход учитывается в `get_monthly_summary`)

### Снятие со счёта (withdrawal)
1. Создаёт `Income(source=account.name, savings_account_id=X, amount, date, description)`
2. Баланс месяца увеличивается автоматически

### Создание счёта
- name (обязательно), color, icon, target_amount (опционально)

### Редактирование счёта
- name, color, icon, target_amount

### Удаление счёта
- Только если нет связанных транзакций (Expense/Income с savings_account_id = X)
- Иначе — ошибка 409

## Маршруты

| Метод | URL | Действие |
|---|---|---|
| GET | `/savings` | Страница счетов + история операций |
| POST | `/savings/add` | Создать счёт (JSON) |
| GET/POST | `/savings/<id>/edit` | Форма редактирования |
| POST | `/savings/<id>/deposit` | Пополнить (JSON: amount, date, description) |
| POST | `/savings/<id>/withdraw` | Снять (JSON: amount, date, description) |
| DELETE | `/savings/<id>` | Удалить счёт |

## UI

### Главный дашборд (`/`)

Блок «Накопительные счета» под строкой баланса:
- Горизонтальные карточки для каждого активного счёта
- На каждой: иконка, название, текущий баланс, `X из Y ₽ — Z%`, прогресс-бар, кнопки «+ Пополнить» / «− Снять»
- Ссылка «Все счета →» ведёт на `/savings`
- Если счетов нет — блок не отображается

### Страница `/savings`

**Верхняя часть:** список карточек счетов
- Иконка, название, количество операций, текущий баланс, целевая сумма
- Прогресс-бар с подписями «0 ₽ / X% достигнуто / цель»
- Кнопки: «+ Пополнить», «− Снять», «✏️ Изменить»
- Кнопка «+ Новый счёт» в заголовке страницы

**Нижняя часть:** история всех операций (все счета)
- Столбцы: иконка типа (↓/↑), описание + название счёта, сумма (зелёная/красная), дата

### Списки расходов и доходов

Транзакции с `savings_account_id != null` помечаются бейджиком `🏦 Название счёта` рядом с описанием.

### Модальные окна пополнения/снятия

Поля:
- Сумма (обязательно, > 0)
- Дата (обязательно, default сегодня)
- Описание (опционально)

При снятии: валидация — нельзя снять больше текущего баланса счёта.

## Обработка ошибок

| Ситуация | Ответ |
|---|---|
| Снятие > баланса счёта | 400 + сообщение с текущим балансом |
| Удаление счёта с транзакциями | 409 + количество транзакций |
| Неверный формат суммы/даты | 400 |
| Чужой счёт | 403 / 404 |

## Out of scope

- Проценты/начисления по счёту
- Переводы между счетами
- Разбивка истории по счёту (фильтр — можно добавить позже)
- Категории для снятий (всегда идут как `source = account.name`)
