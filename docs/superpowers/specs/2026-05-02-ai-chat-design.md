# ИИ-помощник для трекера расходов

## Обзор

Плавающий чат-виджет с ИИ-помощником на базе Ollama (llama3), интегрированный в существующее Flask-приложение. Пользователь может добавлять, редактировать, удалять расходы и доходы, а также запрашивать статистику через естественный язык.

## Технический стек

- **Провайдер:** Groq API (groq.com)
- **Модель:** llama3-8b-8192
- **Метод взаимодействия:** JSON-режим — LLM всегда отвечает структурированным JSON
- **Память:** История из последних 10 сообщений хранится на фронтенде, передаётся с каждым запросом
- **Язык общения:** Русский

## Архитектура

```
Пользователь → Чат-виджет (JS) → POST /api/chat (Flask)
                                        ↓
                              Загружаем контекст пользователя:
                              - список категорий
                              - баланс текущего месяца
                              - последние 5 транзакций
                                        ↓
                              Ollama API (llama3) с:
                              - системным промптом
                              - историей диалога (до 10 сообщений)
                              - сообщением пользователя
                                        ↓
                              Парсим JSON-ответ
                                        ↓
                    Выполняем действие в БД (если action != "none")
                                        ↓
                              Возвращаем {message, action, result}
```

## API

### POST /api/chat

**Запрос:**
```json
{
  "message": "добавь расход 500 на еду",
  "history": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

**Ответ Ollama (JSON-режим):**
```json
{
  "message": "Добавил расход 500 ₽ в категорию «Продукты»",
  "action": "add_expense",
  "params": {
    "amount": 500,
    "category": "Продукты",
    "description": "еда",
    "date": "2026-05-02"
  }
}
```

**Ответ клиенту:**
```json
{
  "message": "Добавил расход 500 ₽ в категорию «Продукты»",
  "ok": true
}
```

## Доступные действия ИИ

| Действие | Параметры | Описание |
|---|---|---|
| `add_expense` | amount, category, description?, date? | Добавить расход |
| `add_income` | amount, source, description?, date? | Добавить доход |
| `get_balance` | — | Текущий баланс и остаток бюджета |
| `get_expenses` | category?, date_from?, date_to? | Список расходов |
| `get_income` | date_from?, date_to? | Список доходов |
| `edit_expense` | expense_id, amount?, category?, description? | Редактировать расход |
| `delete_expense` | expense_id | Удалить расход |
| `edit_income` | income_id, amount?, source?, description? | Редактировать доход |
| `delete_income` | income_id | Удалить доход |
| `none` | — | Только ответить, без действия |

## Системный промпт

Каждый запрос включает:
1. Роль и инструкции по формату JSON
2. Список доступных действий с параметрами
3. Текущий контекст пользователя (категории, баланс)
4. Требование отвечать на русском

## Фронтенд — чат-виджет

### Размещение
Фиксированная плавающая кнопка 🤖 в правом нижнем углу, присутствует на всех страницах (в `base.html`).

### Состояния
- **Закрыт:** только круглая кнопка с градиентом (indigo → purple)
- **Открыт:** попап 360×500px с хедером, лентой сообщений и полем ввода

### Поведение
- Анимация появления снизу вверх при открытии
- Индикатор "печатает..." пока Ollama обрабатывает запрос
- Для `delete_expense` / `delete_income` — диалог подтверждения перед выполнением
- История сессии: до 10 сообщений в памяти JS (сбрасывается при перезагрузке)
- Enter для отправки, Shift+Enter для переноса строки

### Пузыри сообщений
- Сообщения пользователя — справа, фиолетовый фон
- Ответы ИИ — слева, тёмный фон
- Успешные действия — с иконкой ✅
- Ошибки — с иконкой ❌

## Обработка ошибок

- Groq API недоступна → сообщение "ИИ-помощник сейчас недоступен, попробуй позже"
- Невалидный JSON от LLM → повторный запрос с уточнением формата (1 retry)
- Категория не найдена → ИИ предлагает ближайшую из существующих
- Недостаточно данных в запросе → ИИ уточняет недостающие параметры
