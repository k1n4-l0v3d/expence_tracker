# Дизайн: карточка «Можно тратить в день»

## Цель

Показывать пользователю сколько денег он может тратить в день, исходя из текущего баланса и количества дней до ближайшей выплаты (аванс или зарплата).

## Формула

```
daily_budget = balance / days_until_nearest_payment
```

- `balance` — доходы минус расходы за текущий месяц (уже вычисляется в `index`)
- `days_until_nearest_payment` — количество дней от сегодня до ближайшей из двух дат (минимум 1)
- Результат может быть отрицательным — показываем насколько человек уже «в долгу перед собой»

## База данных

Два новых поля в модели `User` (`users`):

| Поле | Тип | Описание |
|---|---|---|
| `salary_day` | Integer, nullable | День месяца зарплаты (1–31) |
| `advance_day` | Integer, nullable | День месяца аванса (1–31) |

Добавляются через `ALTER TABLE users ADD COLUMN` при первом запуске (с проверкой существования колонки).

## Логика вычисления ближайшей даты

```python
def next_payment_date(day: int) -> date:
    today = date.today()
    try:
        candidate = today.replace(day=day)
    except ValueError:
        # день больше чем дней в месяце (например, 31 в феврале)
        candidate = last_day_of_month(today)
    if candidate <= today:
        # переходим на следующий месяц
        next_month = today.replace(day=1) + timedelta(days=32)
        try:
            candidate = next_month.replace(day=day)
        except ValueError:
            candidate = last_day_of_month(next_month)
    return candidate

def get_nearest_payment(salary_day, advance_day) -> tuple[date, str]:
    dates = []
    if salary_day:
        dates.append((next_payment_date(salary_day), 'зарплаты'))
    if advance_day:
        dates.append((next_payment_date(advance_day), 'аванса'))
    return min(dates, key=lambda x: x[0])
```

## UI

### Карточка на главной (index.html)

Четвёртая карточка в строке итогов. Варианты состояния:

**Даты не заданы:**
- Заголовок: «Можно тратить в день»
- Тело: «Укажите даты выплат»
- Подпись: «нажмите, чтобы настроить»
- Стиль: нейтральный серый градиент

**Даты заданы, баланс положительный:**
- Заголовок: «Можно тратить в день»
- Сумма: `+1 234 ₽` (зелёный градиент `grad-balance`)
- Подпись: «до аванса 5 дней» / «до зарплаты 12 дней»

**Даты заданы, баланс отрицательный:**
- Заголовок: «Можно тратить в день»
- Сумма: `-500 ₽` (красный градиент `grad-danger`)
- Подпись: «перерасход · до зарплаты 8 дней»

### Модалка при клике на карточку

Bootstrap-модалка с формой:
- Поле «День аванса» (number, 1–31, необязательное)
- Поле «День зарплаты» (number, 1–31, необязательное)
- Кнопка «Сохранить» → POST `/api/payment-days`
- Кнопка «Отмена»

## API

### GET `/` (index)

Добавить в контекст шаблона:
- `daily_budget` — Float или None (если даты не заданы)
- `nearest_payment_label` — строка «до зарплаты 8 дней» или None
- `salary_day` — Integer или None (для заполнения модалки)
- `advance_day` — Integer или None

### POST `/api/payment-days`

Тело (form-data или JSON):
```json
{ "salary_day": 25, "advance_day": 10 }
```

Валидация: оба поля опциональны, значение 1–31 или null.
Ответ: `{ "ok": true }` или `{ "error": "..." }`.

## Обработка граничных случаев

- Оба поля пустые → карточка в состоянии «не настроено»
- День > последнего дня месяца → берём последний день месяца
- `days_left = 0` (сегодня день выплаты) → используем 1, чтобы не делить на ноль
- Баланс 0 → показываем `0 ₽` без знака

## Что не входит в scope

- Учёт переноса выплат из-за выходных
- История изменения дат выплат
- Разные суммы выплат
